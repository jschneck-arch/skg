## Refined Findings

### 1. Web incompatibility is active across four runtime layers, not one stale file

- daemon web sensor emits `WEB-*` and defaults to `web_surface_v1`
- CLI web observe drives collector runs to `DISCOVERY_DIR/observe_web_*.ndjson`
- active web collector/auth paths emit `WB-*` and default to `web_sqli_to_shell_v1`
- structured fetch emits `WB-30..WB-40` and also defaults to `web_sqli_to_shell_v1`
- checked-in web catalog defines only 4 attack paths and 20 wickets (`WB-01..WB-20`)
- gravity runtime references 34 distinct `WB-*` wickets and generates follow-on proposals for `web_cmdi_to_shell_v1` and `web_sqli_to_shell_v1`
- exploit dispatcher defines additional active web path IDs not present in the checked-in catalog

Conclusion:

The current web split is not just doc drift. It is live across CLI, sensor, gravity, exploit, and toolchain layers.

### 2. Sweep-local projection is not implemented the way the loop claims

- `SensorLoop._auto_project_all()` filters by `since_run_id`
- `project_events_dir()` implements that filter as a filename suffix glob `*_{run_id}.ndjson`
- `emit_events()` writes timestamp/tag filenames with no `run_id`
- the loop therefore falls back to recent full scans

Conclusion:

Per-sweep projection attribution is currently heuristic rather than real.

### 3. `/collect` remains untrustworthy and still lacks focused regression coverage

- `collect_host()` creates a one-target config but the SSH sensor reloads config targets
- `_force_collect` is written but unused
- interval gating still applies in explicit collection mode
- success is reported even when zero events are emitted
- daemon `/collect` still expects a filename pattern the sensor does not produce
- no focused `/collect` or `collect_host()` regression was found in `tests/`

### 4. Authority mixing is explicit in both daemon and topology

- daemon target/world formation merges config, discovery surface, and local injected targets
- topology still imports supplementary `world::*` states from selected discovery surfaces and daemon runtime world snapshots

Conclusion:

Measured-state authority is still only partially enforced.

### 5. Test coverage is stronger on contracts than on integration boundaries

- there is a direct test for `struct_fetch._event()` canonical subject contract
- there is no focused test that proves the default `struct_fetch` path ID resolves against the checked-in web catalog
- there is no focused test for CLI web observe writing to discovery vs measured events
- there is no focused test for `since_run_id` filename filtering
