Conclusion:

The resonance prompt-mode workflow is currently code-present but operator-unreachable.

### 2. Resonance/Ollama tests are much narrower than the feature surface

- `test_ollama_backend.py` only covers config loading
- `test_resonance_drafter.py` only covers the direct Ollama happy path with backend methods patched
- no direct tests were found for backend availability/model-selection/generation/status or for resonance CLI flows

Conclusion:

Resonance backend-selection and fallback semantics are substantially under-tested.

### 3. `test_gravity_routing.py` overstates what it validates

- the file describes itself as an integration routing test with scheduler coverage

Conclusion:

At least part of the routing test boundary is labeled more strongly than it is implemented.

- `OllamaBackend` still mentions configuration coming from `sensors.yaml`
- the actual loader reads `skg_config.yaml`

Conclusion:

The resonance backend is implemented, but parts of its operator/documentation story lag the live code.
