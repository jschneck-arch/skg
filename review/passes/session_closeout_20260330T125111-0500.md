## Refined Findings

### 1. Forge, intel, engagement, and training are real subsystems, not placeholders

- `skg.forge.pipeline` runs gap detection -> generation -> validation -> proposal and emits MC-03 / CP-01 signals
- `skg.intel.engagement_dataset` builds a SQLite telemetry product and runs DP-style integrity analysis over it
- `skg.intel.redteam_to_data` performs cross-domain security -> data integrity derivation
- `skg.training` contains a real corpus, scheduler, generated fine-tune script path, model registration, and eval gate

Conclusion:

The repo map needs to treat these as active secondary-analysis and growth layers, not sidecar experiments.

### 2. Training corpus side effects are duplicated on proposal accept/reject

- `forge.proposals.accept()` calls `on_proposal_accept()` twice
- `forge.proposals.reject()` calls `on_proposal_reject()` twice
- `training.corpus.add_example()` has no dedupe guard

Conclusion:

Accepted and rejected proposal examples are likely being double-counted in the corpus.

### 3. Operator-facing CLI exposure lags implemented modules

- `engagement_dataset` is wired into `skg engage`
- `redteam_to_data.py` advertises `skg data redteam`, but `cmd_data()` and `cli.app` expose no such subcommand
- `training.scheduler` advertises `skg train run`, but `cli.app` exposes no `train` command

Conclusion:

Part of the repo's implemented analysis surface is only reachable by direct module invocation, not by the advertised CLI.

### 4. Training packaging path is present but operationally weak

- `scripts/skg-train.service` is templated with `User=%i`
- `scripts/skg-train.timer` targets plain `skg-train.service`

Conclusion:

The packaged daily training path is unlikely to work cleanly without manual instance wiring.

### 5. Test coverage is asymmetric

- `surface`, `forge` proposal/pipeline behavior, and `gap_detector` have at least some real coverage
- `engagement_dataset` only surfaced here through CLI mocking coverage
- no direct tests were found for `skg.training.corpus`, `skg.training.scheduler`, or `skg.training.trainer`

Conclusion:

The newer learning and engagement-analysis layers are implemented, but not validated to the same standard as the core substrate/surface paths.
