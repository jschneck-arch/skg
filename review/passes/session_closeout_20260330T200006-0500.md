## Findings Added This Slice

1. `kernel.adapters` still depends on filename shape too early, so identity-aware observation loading can miss discovery artifacts for hostname aliases.
2. The gravity/helper alias seam extends past label handling into actual scoring: current `BA-*`, `DP-*`, and `IF-*` families can miss sphere inference and domain-hint boosts.
3. The default inter-local coupling tables still privilege older domain names numerically, so current domain names can receive materially weaker coupling unless config overrides them.
4. Current test coverage still pins several old `data` / `container` sphere names in topology and coupling paths, so it will not reliably catch current-name regressions.
