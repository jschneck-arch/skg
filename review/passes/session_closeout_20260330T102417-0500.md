## New Concrete Findings

1. `collect_host()` can return success on zero events.
   - It returns `True` after `sensor.run()` without checking whether any events were emitted.

2. Single-target collection still ignores the supplied target and uses incompatible artifact naming.
   - `collect_host()` supplies one target.
   - `SshSensor.run()` reloads config targets.
   - `/collect` then looks for a non-existent deterministic event filename pattern.

3. The live web runtime path is more broken than originally recorded.
   - Wicket namespace mismatch: `WEB-*` vs `WB-*`
   - Attack-path mismatch: `web_surface_v1` and `web_sqli_to_shell_v1` vs catalog `web_initial_access_v1`
   - Alias mismatch: `web_sqli_to_shell_v1 -> web_full_chain_v1`, but `web_full_chain_v1` is absent from the checked-in catalog
   - Stale root projector still exists and points at a non-existent catalog file

4. The topology authority split is explicit in code.
   - `compute_field_energy_all()` merges event/interp state, then appends realized `world::*` states from both discovery surfaces and daemon world snapshots.
   - This is not just an accidental read path; it is codified behavior.

5. The supply-chain legacy layer appears effectively isolated.
   - Current tests and active toolchain/runtime paths use `SC-*`.
   - `PKG-*` references appear confined to `skg.domains.supply_chain`.

6. Bootstrap coverage is inconsistent with the current domain registry.
   - `install_layer4.sh` installs only a subset of registered toolchains and bootstraps an even smaller subset.
   - `setup_arch.sh` covers more, but still not the full registered inventory.
